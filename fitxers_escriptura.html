Escriptura en fitxers de text

Un cop hem vist l'operació de lectura en fitxers de text, la d'escriptura no ens sorprendrà gaire.

El conegut patró obrir→usar→tancar ara es concreta a obrir→escriure→tancar

El que és escrit, roman

Comencem pel més bàsic. Considera aquest codi:

 1/*
 2 * Programa que demostra una escriptura bàsica en fitxer de text
 3 */
 4import java.io.IOException;
 5import java.io.FileWriter;
 6import java.io.BufferedWriter;
 7public class EscripturaBasica {
 8    public static void main(String[] args) throws IOException {
 9        String cami = "resultat.txt";
10        String text = "Verba volant, scripta manent";
11        BufferedWriter sortida = new BufferedWriter(new FileWriter(cami));  // obrir
12        sortida.write(text);                                                // escriure
13        sortida.close();                                                    // tancar
14    }
15}

Veiem-ne una execució:

$ javac EscripturaBasica.java && java EscripturaBasica
$ cat resultat.txt
Verba volant, scripta manent

Doncs sí, sembla que el nostre programa ha escrit un text al fitxer resultat.txt.

Experimentem una mica més:

$ rm resultat.txt                               # eliminem el fitxer
$ java EscripturaBasica                         # tornem a executar
$ cat resultat.txt
Verba volant, scripta manent
$ echo "Text que no romandrà" > resultat.txt    # posem un altre text
$ cat resultat.txt
Text que no romandrà
$ java EscripturaBasica                         # tornem a executar
$ cat resultat.txt
Verba volant, scripta manent
$ java EscripturaBasica                         # un altre cop!
$ cat resultat.txt
Verba volant, scripta manent

A l'experiment anterior podem veure el següent comportament:

  • si el fitxer no existeix, el crea

  • si el fitxer existeix sobreescriu o, si vols, reemplaça el seu contingut

  • si l'executem més d'un cop seguit, no es distingeix cap canvi

Salts de línia

Si hem escrit una línia, bé en podrem escriure dues, oi?

 1/*
 2 * Programa que demostra una escriptura de dues línies en fitxer de text
 3 */
 4import java.io.IOException;
 5import java.io.FileWriter;
 6import java.io.BufferedWriter;
 7public class EscripturaDuesLinies {
 8    public static void main(String[] args) throws IOException {
 9        String cami = "resultat.txt";
10        String primeraLinia = "Babau beu vi bo i viu bé.";
11        String segonaLinia = "En cap cap cap el que cap en aquest cap.";
12        BufferedWriter sortida = new BufferedWriter(new FileWriter(cami));
13        sortida.write(primeraLinia);
14        sortida.write(segonaLinia);
15        sortida.close();
16    }
17}

En executar aquest nou programa, però, ens trobem amb una no tan agradable sorpresa:

$ javac EscripturaDuesLinies.java && java EscripturaDuesLinies
$ cat resultat.txt
Babau beu vi bo i viu bé.En cap cap cap el que cap en aquest cap.

Ops! Les dues línies s'han quedat… a la mateixa línia!

El procediment write() dels BufferedWriter no ens escriu el salt de línia per nosaltres.

De fet, et confesso que he fet una mica de trampes i t'he maquillat la sortida de la comanda cat de manera que no es veiés que el prompt de la meva shell apareix tot just seguint el resultat del cat.

Per solucionar aquest problema podem fer escriure el caràcter de salt de línia '\n'. És una solució fàcil que te l'inconvenient de generar un resultat no compatible amb alguns sistemes operatius.

Per solucionar aquest problema hi ha diferents solucions. Per exemple, reemplaçar '\n' per System.lineSeparator() o, si fem servir String.format() podem utilitzar el %n. Per exemple:

sortida.write(String.format("Babau beu vi bo i viu bé.%n"));

Et proposo una altra de ben senzilla que ens ofereix BufferedWriter:

 1/*
 2 * Programa que demostra una escriptura de dues línies en fitxer de text
 3 * Amb salt de línia preparat pel S.O.
 4 */
 5import java.io.IOException;
 6import java.io.FileWriter;
 7import java.io.BufferedWriter;
 8public class EscripturaDuesLinies {
 9    public static void main(String[] args) throws IOException {
10        String cami = "resultat.txt";
11        String primeraLinia = "Babau beu vi bo i viu bé.";
12        String segonaLinia = "En cap cap cap el que cap en aquest cap.";
13        BufferedWriter sortida = new BufferedWriter(new FileWriter(cami));
14        sortida.write(primeraLinia);
15        sortida.newLine();
16        sortida.write(segonaLinia);
17        sortida.newLine();
18        sortida.close();
19    }
20}

Per a aquest curs, fes servir l'opció que més t'agradi.

Ampliació de fitxers

Amb el que hem vist fins ara, cada cop que obrim un fitxer per escriptura, si existia perdem tot el que hi havia. Què passaria si volguéssim afegir noves línies a un fitxer existent, sense perdre les antigues?

Una opció seria copiar el fitxer original a un altre lloc de manera que puguem llegir les línies originals. A continuació sobreescriuríem el fitxer original tot reescrivint les línies a partir de la còpia i, abans de tancar, afegint les noves línies. Finalment hauríem d'eliminar el fitxer còpia per deixar-ho tot net. No sembla gaire eficient, oi?

Podem resoldre aquest problema molt més fàcilment obrint el fitxer en el que es coneix com a mode ampliació. En Java ho podem fer quan inicialitzem el nostre FileWriter.

 1import java.io.IOException;
 2import java.io.FileWriter;
 3import java.io.BufferedWriter;
 4public class Ampliant {
 5    public static void main(String[] args) throws IOException {
 6        String cami = "resultat.txt";
 7        BufferedWriter sortida = new BufferedWriter(new FileWriter(cami, true));
 8        //                          El mode ampliació s'especifica aquí  ↑↑↑↑
 9        sortida.write("Bai Maitea, bai.\n");
10        sortida.close();
11    }
12}

El punt important és que a new FileWriter() li passem un true. Només això!

Trobaràs els detalls a la documentació oficial

Considera la següent execució:

$ echo "M'has sentit?" > resultat.txt   # Creem un fitxer amb un contingut inicial
$ cat resultat.txt
M'has sentit?
$ java Ampliant                         # Hi afegim una línia
$ cat resultat.txt
M'has sentit?
Bai Maitea, bai.
$ java Ampliant                         # Hi afegim una línia més
$ cat resultat.txt
M'has sentit?
Bai Maitea, bai.
Bai Maitea, bai.
