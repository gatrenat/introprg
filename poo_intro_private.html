Propietats protegides

Considera la següent implementació de GatRenat:

1public class GatRenat {
2    public int vides = 7;
3}

Aquesta declaració ens permet fer el següent:

1public class UsaGatRenat {
2    public static void main(String[] args) {
3        GatRenat renat = new GatRenat();
4        renat.vides = -12;
5        System.out.println("El gat Renat té " + renat.vides + " vides");
6    }
7}

Que UsaGatRenat funcioni no ens hauria de deixar massa tranquil·litat. -12 de vida? Potser per un joc de zombis podria ser vàlid però en general no sembla quelcom gaire desitjable.

El que voldríem és aconseguir evitar que ens puguin assignar un valor no vàlid per les vides del gat Renat.

No et pensis que és quelcom a l'abast de tots els llenguatges de programació. Per Java, però, veurem que és força senzill.

T'has percatat del símbol - davant de vides al diagrama?

Considera aquesta nova versió de GatRenat:

 1public class GatRenat {
 2    private int vides = 7;
 3    public int getVides() {  //  retorna el nombre de vides
 4        return vides;
 5    }
 6    public void setVides(int novesVides) {   // modifica el nombre de vides si ens donen un de vàlid
 7        if (novesVides >= 0) {
 8            vides = novesVides;
 9        }
10    }
11}

Contesta les següents preguntes:

  1. Quins canvis trobes al codi anterior?

  2. Què passa si intentes executar el següent codi? Perquè creus?

    1public class UsaGatRenat {
2    public static void main(String[] args) {
3        GatRenat renat = new GatRenat();
4        System.out.println("El gat Renat té " + renat.vides + " vides");
5    }
6}

  3. Què tal si modifiquem lleugerament el codi anterior per aquest?

    1 public class UsaGatRenat {
2     public static void main(String[] args) {
3         GatRenat renat = new GatRenat();
4         System.out.println("El gat Renat té " + renat.getVides() + " vides");
5     }
6 }

    Fixa't que ara estem fent servir renat.getVides() en comptes de directament renat.vides(). Funciona ara?

  4. Com es pot determinar a UsaGatRenat que renat té 5 vides?

    Inclou el codi a la resposta i la sortida que et genera.

  5. Què passa si intentem assignar-li -12 en comptes de 5 vides?

    Inclou el codi, la sortida i la teva explicació.

  6. Un cop hem vist què passa des de UsaGatRenat toca experimentar com es comporta aquest private des del main() del propi GatRenat.

    Experimenta afegint a la darrera versió de GatRenat la mateixa declaració de main() que apareixia a la primera versió de UsaGatRenat d'aquesta pàgina. És a dir:

     1public class GatRenat {
 2    private int vides = 7;
 3    public int getVides() {  //  retorna el nombre de vides
 4        return vides;
 5    }
 6    public void setVides(int novesVides) {   // modifica el nombre de vides si ens donen un de vàlid
 7        if (novesVides >= 0) {
 8            vides = novesVides;
 9        }
10    }
11    public static void main(String[] args) {
12        GatRenat renat = new GatRenat();

    Què passa en intentar compilar i executar aquesta versió? Ho veus coherent? Tens alguna explicació que ho justifiqui?

  7. Considerant aquesta darrera versió de GatRenat, és possible aconseguir assignar un nombre de vides negatiu a una instància de GatRenat sense modificar el codi de la classe GatRenat? Perquè o com?

  8. Com descriuries el paper de public i private a les propietats d'una classe?

  9. Aquesta implementació de GatRenat disposa de getter i de setter. T'has plantejat què passaria si només en tingués getter? i si només en tingués setter? i si no en tingués cap dels dos?

  10. Per cert, t'has fixat que els mòduls getVides() i setVides() tenen una definició lleugerament diferent als mòduls que hem declarat fins ara en aquest curs? En concret, no els falta alguna cosa?

    Si no se t'acut, considera revisar quan varem veure per primer cop un mòdul que no fos el main().

    De moment només cal que identifiquis la petita diferència en la declaració, però si tens molta curiositat, experimenta què passa si declares getVides() de la mateixa manera que dibuixaQuadrat().

Una mica de culturilla POO

Als mòduls getVides() i setVides() en orientació a objectes els anomenem accessors doncs ens permeten accedir a propietats privades, en aquest cas vides. De fet, aquesta manera d'anomenar els accessors és tan típica que sovint parlem de getters i setters.
