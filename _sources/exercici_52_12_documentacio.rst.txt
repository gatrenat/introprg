.. rubric:: Enunciat

En aquest exercici començarem a documentar els canvis que realitzarem al nostre
projecte.

Crea un document una mica formal (amb portada, taula de continguts, secció
d'introducció i conclusions) i afegeix una secció on incloguis el diagrama de
:doc:`classes <refactoring_exemple_partida>`, i el de
:doc:`seqüència<refactoring_exemple_informe>` d'inici del projecte de refacció.

Pots aprofitar la secció de conclusions per descriure les teves reflexions
respecte l'estat actual del projecte.

Aquest document l'aniràs completant a mida que vagis avançant en els exercicis.

Respecte els diagrames
======================

Per realitzar els diagrames, pots fer servir qualsevol eina de dibuix, però et
recomano que li facis una ullada a `plantuml <https://plantuml.com/>`_. Una
petita inversió aquí, et valdrà la pena.

Per exemple, el diagrama de classes, el representaries:

.. code-block:: none

    class Vehicle {
        + {static} BASIC = 1: int
        + {static} GENERAL = 2: int
        + {static} LUXE = 3: int
        - model: String
        - marca: String
        - categoria: int
        + Vehicle(marca, model, categoria)
    }

    class Lloguer {
        - dies: int
        + Lloguer(vehicle, dies)
    }

    class Client {
        - nif: String
        - nom: String
        - telèfon: String
        + informe(): String
    }

    Vehicle <-left- Lloguer
    Lloguer <-left- Client: *



Per la seva banda, el diagrama de seqüència el podries descriure amb:

.. code-block:: none

    actor Usuari
    Usuari -> unClient: informe()
    unClient -> unLloguer: getVehicle()
    note left: per cada lloguer
    unClient -> unVehicle: getMarca()
    unClient -> unVehicle: getModel()
    unClient -> unVehicle: getCategoria()
    unClient -> unLloguer: getDies()
