Enumerats

Fins ara hem fet servir literals de tipus String per definir els diferents valors d'algunes propietats. Per exemple, recordem el nostre amic el Gat Renat:

Recordem que els valors vàlids de posicio eren "dret", "assegut" i "estirat". i que, per a assegurar que sempre eren vàlids, havíem de controlar els valors en els diferents punts en que es podien canviar: al constructor i al setter

 1 public class GatRenat {
 2     private String posicio = "estirat";
 3     public GatRenat(String posicio) {
 4         setPosicio(posicio);
 5     }
 6     public void setPosicio(String posicio) {
 7         if posicioValida(posicio) {
 8             this.posicio = posicio;
 9         }
10     }
11     public static boolean posicioValida(String posicio) {
12         return  "dret".equals(posicio) || "assegut".equals(posicio) || "estirat".equals(posicio);
13     }
14     /* … */
15 }

D'aquesta manera, podem estar segurs que l'estat del gat Renat sempre tindrà un valor correcte per posicio, fins i tot quan li assignem un valor que no coneixem en temps de compilació:

1 GatRenat renat = new GatRenat(Entrada.readLine());

Si bé aquest mecanisme funciona i ens garanteix que posicio només pot tenir valors vàlids, no és gaire satisfactori. Un problema interessant és que, en cas que li intentem assignar un valor no vàlid des de programa, l'error només serà detectable en el moment de l'execució.

1 renat.setEstat("assagut");     // atenció a la segona 'a'!

No hi haurà algun mecanisme per enumerar tots els valors vàlids de posicio?

enum

Com molts llenguatges de programació, Java ens ofereix la possibilitat de definir enumerats: tipus de dades que tenen limitats els valors possibles a una llista.

En el cas de Java, aquesta funcionalitat l'ofereixen els enum.

Una bona font de consulta és el tutorial de enumerats "oficial".

Si mirem aquesta documentació/tutorial, veurem que ens ofereix la següent definició:

1 public enum DiaSetmana {
2     DILLUNS, DIMARTS, DIMECRES, DIJOUS, DIVENDRES, DISSABTE, DIUMENGE
3 }

Un cop definit DiaSetmana, podem tenir variables d'aquest tipus, assignar-li valors, etc:

1DiaSetmana dia = DiaSetmana.DILLUNS;
2
3if (dia.equals(DiaSetmana.DIVENDRES)) {
4   System.out.println("Aquesta nit, festa de teulades!");
5}

enum és una classe

Els enumerats en Java són en realitat un tipus especial de classe que indica les instàncies possibles.

Com una classe qualsevol, té la possibilitat de definir els seus propis atributs i mètodes.

Per exemple, considera el següent codi:

 1 public class UsaDiaSetmana {
 2     public static void main(String[] args){
 3         for (DiaSetmana dia: DiaSetmana.values()) {
 4             System.out.printf("El %s està en la posició %d%n", dia, dia.getCodi());
 5         }
 6
 7         // obtenim un dia des del seu codi
 8         int diaSonInt = 1;
 9         DiaSetmana diaSon = DiaSetmana.fromCodi(diaSonInt);
10         System.out.printf("Els %s tenim molta són%n", diaSon);
11
12         /// obtenim un dia des del seu nom
13         String diaFestaStr = "divendres";
14         DiaSetmana diaFesta = DiaSetmana.fromString(diaFestaStr);
15         System.out.printf("Els %s fem festa a la teulada%n", diaFesta);
16
17         // veiem què passa si demanem un nom que no existeix
18         try {
19             DiaSetmana.fromString("un dia qualsevol");
20         } catch (IllegalArgumentException e) {
21             System.out.println("Capturada la següent excepció:");
22             e.printStackTrace();
23         }
24
25         // una manera similar d'aconseguir el mateix que abans
26         try {
27             DiaSetmana.valueOf("un dia qualsevol");
28         } catch (IllegalArgumentException e) {
29             System.out.println("Capturada la següent excepció:");
30             e.printStackTrace();
31         }
32     }
33 }
34
35 interface Codificat {
36     public int getCodi();
37 }
38
39 enum DiaSetmana implements Codificat {
40     DILLUNS   (1),
41     DIMARTS   (2),
42     DIMECRES  (3),
43     DIJOUS    (4),
44     DIVENDRES (5),
45     DISSABTE  (6),
46     DIUMENGE  (7);
47
48     private int codi;
49
50     private DiaSetmana(int codi) { this.codi = codi; }
51
52     public int getCodi() { return codi; }
53
54     @Override
55     public String toString() { return super.toString().toLowerCase(); }
56
57     public static DiaSetmana fromCodi(int codi) {
58         for (DiaSetmana dia: DiaSetmana.values()) {
59             if (dia.getCodi() == codi) return dia;
60         }
61         throw new IllegalArgumentException(String.format("Codi %d no suportat per DiaSetmana", codi));
62     }
63
64     public static DiaSetmana fromString(String nom) {
65         for (DiaSetmana dia: DiaSetmana.values()) {
66             if (dia.name().equalsIgnoreCase(nom)) return dia;
67         }
68         throw new IllegalArgumentException(String.format("Nom \"%s\" no suportat per DiaSetmana", nom));
69     }
70 }

T'has fixa't? Al final, Java ha traduït enum a una classe més o menys normaleta? Permet-me que et remarqui uns punts:

  • ofereix una propietat estàtica i final per cada valor possible de l'enumerat

  • ofereix un array que conté cadascun dels valors

  • la classe està declarada amb el mateix nom que l'enumerat, i marcada com a final. Aquest modificador, quan està associat a una classe, fa que aquesta no pugui ser estesa (no podrà tenir subclasses)
