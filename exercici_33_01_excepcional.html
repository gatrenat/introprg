Exercici 33_01. Situacions excepcionals

Context

  • Carpeta de lliurament: 33_01_excepcional/

  • Continguts relacionats: Lectura de fitxers de text

  • Com lliurar-lo: instruccions

Enunciat

Considera el següent codi novament:

 1/*
 2 * programa que mostra el seu propi codi
 3 * Nota: pressuposa que el codi font es troba en la mateixa carpeta que
 4 * l'executable.
 5 */
 6import java.io.BufferedReader;
 7import java.io.FileReader;
 8import java.io.IOException;
 9public class Mostrat {
10    public static void main(String[] args) throws IOException {
11        String cami = "Mostrat.java";
12        FileReader fileReader = new FileReader(cami);
13        BufferedReader input = new BufferedReader(fileReader);
14        while (true) {
15            String linia = input.readLine();
16            if (null == linia) break;
17            System.out.println(linia);
18        }
19        input.close();
20    }
21}

Per realitzar aquest exercici, hauràs de realitzar algunes trapelleries al programa.

Completa aquesta plantilla tot reemplaçant les XXX per la teva resposta:

===========================
Trapelleries a Mostrat.java
===========================

Resum de trapelleries al codi de ``Mostrat.java``


Trapelleria 1. Fitxer inexistent
================================

Canvia el nom del fitxer de la línia 11 per ``inexistent.txt``. Compila
i intenta executar el programa.

L'error que genera és: ::

    $ java Mostrat
    XXX

Trapelleria 2. Fitxer protegit
==============================

Crea ara un fitxer anomenat ``existent.txt`` però protegeix-lo perquè
no pugui ser llegit.

Aquí tens unes instruccions que t'ajudaran: ::

    $ touch existent.txt
    $ chmod -r existent.txt

L'error que genera és: ::

    $ java Mostrat
    XXX


Trapelleria 3. Un directori en comptes d'un fitxer
==================================================

Canvia el nom de ``existent.txt`` per ``/tmp``. Compila i torna a
intentar executar-ho i copia aquí el que et genera: ::

    $ java Mostrat
    XXX
